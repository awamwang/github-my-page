Githug解题方法

发表于 2018-07-18 | 分类于 开发工具 ， 代码管理-Git | |

Githug是一个git练习游戏
https://segmentfault.com/a/1190000004222489
https://github.com/Gazler/githug

通用操作

首先，设置一个自己习惯的编辑器

其次，如果需要的话，配置一些别名，会方便些

攻略

基于v0.5.0

  • Githug *

No githug directory found, do you wish to create one? [yn] y
Welcome to Githug!

  • 1

Name: init
Level: 1
Difficulty: *

A new directory, git_hug, has been created; initialize an empty repository in it.

  • 2

Name: config
Level: 2
Difficulty: *

Set up your git name and email, this is important so that your commits can be identified.

  • 3

Name: add
Level: 3
Difficulty: *

There is a file in your folder called README, you should add it to your staging area
Note: You start each level with a new repo. Don’t look for files from the previous one.

  • 4

Name: commit
Level: 4
Difficulty: *

The README file has been added to your staging area, now commit it.

  • 5

Name: clone
Level: 5
Difficulty: *

Clone the repository at https://github.com/Gazler/cloneme.

  • 6

Name: clone_to_folder
Level: 6
Difficulty: *

Clone the repository at https://github.com/Gazler/cloneme to my_cloned_repo.

  • 7

Name: ignore
Level: 7
Difficulty: **

The text editor ‘vim’ creates files ending in .swp (swap files) for all files that are currently open. We don’t want them creeping into the repository. Make this repository ignore .swp files.

在最后插入*.swp

  • 8

Name: include
Level: 8
Difficulty: **

Notice a few files with the ‘.a’ extension. We want git to ignore all but the ‘lib.a’ file.

在最后插入

  • 9

Name: status
Level: 9
Difficulty: *

There are some files in this repository, one of the files is untracked, which file is it?

  • 10

Name: number_of_files_committed
Level: 10
Difficulty: *

There are some files in this repository, how many of the files will be committed?

  • 11

Name: rm
Level: 11
Difficulty: **

A file has been removed from the working tree, however the file was not removed from the repository. Find out what this file was and remove it.

  • 12

Name: rm_cached
Level: 12
Difficulty: **

A file has accidentally been added to your staging area, find out which file and remove it from the staging area. NOTE Do not remove the file from the file system, only from git.

  • 13

Name: stash
Level: 13
Difficulty: **

You’ve made some changes and want to work on them later. You should save them, but don’t commit them.

  • 14

Name: rename
Level: 14
Difficulty: ***

We have a file called oldfile.txt. We want to rename it to newfile.txt and stage this change.

  • 15

Name: restructure
Level: 15
Difficulty: ***

You added some files to your repository, but now realize that your project needs to be restructured. Make a new folder named src and using Git move all of the .html files into this folder.

  • 16

Name: log
Level: 16
Difficulty: **

You will be asked for the hash of most recent commit. You will need to investigate the logs of the repository for this.

  • 17

Name: tag
Level: 17
Difficulty: **

We have a git repo and we want to tag the current commit with new_tag.

  • 18

Name: push_tags
Level: 18
Difficulty: **

There are tags in the repository that aren’t pushed into remote repository. Push them now.

  • 19

Name: commit_amend
Level: 19
Difficulty: **

The README file has been committed, but it looks like the file forgotten_file.rb was missing from the commit. Add the file and amend your previous commit to include it.

  • 20

Name: commit_in_future
Level: 20
Difficulty: **

Commit your changes with the future date (e.g. tomorrow).

  • 21

Name: reset
Level: 21
Difficulty: **

There are two files to be committed. The goal was to add each file as a separate commit, however both were added by accident. Unstage the file to_commit_second.rb using the reset command (don’t commit anything).

  • 22

Name: reset_soft
Level: 22
Difficulty: **

You committed too soon. Now you want to undo the last commit, while keeping the index.

  • 23

Name: checkout_file
Level: 23
Difficulty: ***

A file has been modified, but you don’t want to keep the modification. Checkout the config.rb file from the last commit.

  • 24

Name: remote
Level: 24
Difficulty: **

This project has a remote repository. Identify it.

  • 25

Name: remote_url
Level: 25
Difficulty: **

The remote repositories have a url associated to them. Please enter the url of remote_location.

  • 26

Name: pull
Level: 26
Difficulty: **

You need to pull changes from your origin repository.

  • 27

Name: remote_add
Level: 27
Difficulty: **

Add a remote repository called origin with the url https://github.com/githug/githug

  • 28

Name: push
Level: 28
Difficulty: ***

Your local master branch has diverged from the remote origin/master branch. Rebase your commit onto origin/master and push it to remote.

  • 29

Name: diff
Level: 29
Difficulty: **

There have been modifications to the app.rb file since your last commit. Find out which line has changed.

  • 30

Name: blame
Level: 30
Difficulty: **

Someone has put a password inside the file config.rb find out who it was.

  • 31

Name: branch
Level: 31
Difficulty: *

You want to work on a piece of code that has the potential to break things, create the branch test_code.

  • 32

Name: checkout
Level: 32
Difficulty: **

Create and switch to a new branch called my_branch. You will need to create a branch like you did in the previous level.

  • 33

Name: checkout_tag
Level: 33
Difficulty: **

You need to fix a bug in the version 1.2 of your app. Checkout the tag v1.2.

  • 35

Name: branch_at
Level: 35
Difficulty: ***

You forgot to branch at the previous commit and made a commit on top of it. Create branch test_branch at the commit before the last.

  • 36

Name: delete_branch
Level: 36
Difficulty: **

You have created too many branches for your project. There is an old branch in your repo called ‘delete_me’, you should delete it.

  • 37

Name: push_branch
Level: 37
Difficulty: **

You’ve made some changes to a local branch and want to share it, but aren’t yet ready to merge it with the ‘master’ branch. Push only ‘test_branch’ to the remote repository

  • 38

Name: merge
Level: 38
Difficulty: **

We have a file in the branch ‘feature’; Let’s merge it to the master branch.

  • 39

Name: fetch
Level: 39
Difficulty: **

Looks like a new branch was pushed into our remote repository. Get the changes without merging them with the local repository

  • 40

Name: rebase
Level: 40
Difficulty: **

We are using a git rebase workflow and the feature branch is ready to go into master. Let’s rebase the feature branch onto our master branch.

  • 41

Name: rebase_onto
Level: 41
Difficulty: **

You have created your branch from wrong_branch and already made some commits, and you realise that you needed to create your branch from master. Rebase your commits onto master branch so that you don’t have wrong_branch commits.

  • 42

Name: repack
Level: 42
Difficulty: **

Optimise how your repository is packaged ensuring that redundant packs are removed.

  • 43

Name: cherry-pick
Level: 43
Difficulty: ***

Your new feature isn’t worth the time and you’re going to delete it. But it has one commit that fills in README file, and you want this commit to be on the master as well.

  • 44

Name: grep
Level: 44
Difficulty: **

Your project’s deadline approaches, you should evaluate how many TODOs are left in your code

  • 45

Name: rename_commit
Level: 45
Difficulty: ***

Correct the typo in the message of your first (non-root) commit.

  • 46

Name: squash
Level: 46
Difficulty: **

You have committed several times but would like all those changes to be one commit.

  • 47

Name: merge_squash
Level: 47
Difficulty: ***

Merge all commits from the long-feature-branch as a single commit.

  • 48

Name: reorder
Level: 48
Difficulty: **

You have committed several times but in the wrong order. Please reorder your commits.

  • 49

Name: bisect
Level: 49
Difficulty: ***

A bug was introduced somewhere along the way. You know that running ruby prog.rb 5 should output 15. You can also run make test. What are the first 7 chars of the hash of the commit that introduced the bug.

  • 50

Name: stage_lines
Level: 50
Difficulty: **

You’ve made changes within a single file that belong to two different features, but neither of the changes are yet staged. Stage only the changes belonging to the first feature.

  • 51

Name: find_old_branch
Level: 51
Difficulty: **

You have been working on a branch but got distracted by a major issue and forgot the name of it. Switch back to that branch.

  • 52

Name: revert
Level: 52
Difficulty: **

You have committed several times but want to undo the middle commit.
All commits have been pushed, so you can’t change existing history.

  • 53

Name: restore
Level: 53
Difficulty: **

You decided to delete your latest commit by running git reset --hard HEAD^. (Not a smart thing to do.) You then change your mind, and want that commit back. Restore the deleted commit.

  • 54

Name: conflict
Level: 54
Difficulty: **

You need to merge mybranch into the current branch (master). But there may be some incorrect changes in mybranch which may cause conflicts. Solve any merge-conflicts you come across and finish the merge.

  • 55

Name: submodule
Level: 55
Difficulty: **

You want to include the files from the following repo: https://github.com/jackmaney/githug-include-me into a the folder ./githug-include-me. Do this without cloning the repo or copying the files from the repo into this repo.

  • 56

Name: contribute
Level: 56
Difficulty: ***

This is the final level, the goal is to contribute to this repository by making a pull request on GitHub. Please note that this level is designed to encourage you to add a valid contribution to Githug, not testing your ability to create a pull request. Contributions that are likely to be accepted are levels, bug fixes and improved documentation.
